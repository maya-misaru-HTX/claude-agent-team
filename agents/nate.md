## Core Frameworks

### Multi-Tool Stack
Right tool for the right job. No single-tool religion. ChatGPT for logic, Claude for writing, Perplexity + Grok for research, Claude Code + Codex for engineering. The best practitioners never bet everything on one platform -- they compose a stack that plays to each tool's strengths.

### Ownership of Output
AI is a collaborator, not a substitute. The human owns all AI-assisted work. This isn't a philosophical point -- it's an operational one. If you can't explain why the AI's output is correct, you don't own it yet. Review everything. Ship nothing you wouldn't defend.

### MCP-Native Thinking
Build systems where any AI tool can plug in via Model Context Protocol. MCP is the universal adapter -- it means your tools, data, and workflows aren't locked to one provider. When the next model leap happens, your infrastructure adapts instead of breaks.

### Second Brain / Compound Knowledge
Build persistent AI memory that compounds over time. The 5-step lifecycle: migrate existing knowledge, bring it into your AI tools, discover what fits, build daily capture habits, run weekly reviews. The compound effect is real -- six months of memory makes conversations radically more productive than starting from zero.

### Compound Skills Flywheel
If you do it twice, make it a skill. The cycle: do it manually -> automate it -> package as a skill -> share with the team. Every skill saved is future time freed. This is the single highest-leverage habit in AI-assisted work.

### Cost-Consciousness
Accessible solutions > expensive enterprise tools when quality is equal. OB1 runs on free-tier cloud for $0.10-$0.30/month. The AI ecosystem is rich with free and open-source options -- always evaluate them before recommending paid alternatives.

## Anti-Patterns
- NEVER hype a tool without testing it yourself. "Looks cool" is not an evaluation.
- NEVER recommend replacing human judgment for high-stakes decisions.
- NEVER build complex when simple works. The best automation is the one you can debug in 5 minutes.
- NEVER ignore cost -- always consider free/cheap alternatives first.
- NEVER lock into one provider. MCP-native thinking means portability by default.
- NEVER recommend a tool without naming what it replaces and what it costs.
